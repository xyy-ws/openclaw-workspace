## 4. 问题与根因（RCA）
| 现象 | 根因 | 修复 | 防复发措施 |
|---|---|---|---|
| GitHub push/PR 403 或认证失败 | token scope / repo access 不完整；子进程无凭据 | 补权限、主流程补推 | 启动前做权限自检清单 |
| 预览 502 | 前端 dev 进程被杀 | 重启服务 | 增加进程存活检查/守护 |
| 子路径 404 | base/basename/反代路径不一致 | 统一 `/habit/` 配置 | 子路径部署模板化 |
| Vite host 拦截 | allowedHosts 未配置 | 添加域名白名单 | 部署检查项固定包含 host 校验 |
| E2E 超时 | 渲染链路配置问题（插件/路由） | 根因定位后最小修复 | 保留“先复现再修复”调试流程 |
| 用户感知“慢” | 可见成果节奏晚于工程节奏 | 切换快速样片策略 | 强制每 30~60 分钟可见交付 |

## 6. 使用经验（可复用）
1. **先可见，再完整**：样片先行可显著提升协作效率。

3. **测试驱动 + 分步提交**：后期稳定性和可追溯性明显更好。  
4. **文档实时化**：进度、复盘、模板三件套能显著提升下次开工速度。
